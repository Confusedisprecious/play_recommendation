# 社交媒体数据接入

网站使用两条独立管道：

- 白名单公开资讯：`scripts/discover_social.py` 每 2 小时检查一次新闻 RSS。
- 授权社交内容：在本机使用 `xiaohongshu-mcp`、`redbook`、`Agent-Reach`
  或 `MediaCrawler` 搜索，再由 `scripts/import_social_exports.py` 脱敏导入候选队列。

两条管道最终由 `scripts/social_ingest.py` 生成前端使用的
`data/social_feed.json`。

## 当前接入方式

- 小红书：支持 `xiaohongshu-mcp`、`redbook`、`Agent-Reach` 的本地 JSON
  导出，能够按“最新、一天内、一周内、同城、图文/视频”搜索。
- 抖音：支持 `MediaCrawler` 等本地授权工具的 JSON 导出，以及已核验的公开内容页。
- 微博：预留官方 API 接入，需申请应用并提供合法凭证后启用。

登录态只能留在本机工具中，不提交 Cookie、不处理验证码、不复制整篇笔记，
也不下载或重新托管用户视频。Git 仓库只保存公开标题、短摘要、作者、发布时间、
来源链接、封面链接和互动量快照。

## 导入授权搜索结果

先把搜索结果保存到被 `.gitignore` 排除的 `data/imports/`。例如使用 `redbook`：

```powershell
redbook search "武汉 周末去哪玩" --sort latest --json |
  Out-File -Encoding utf8 data/imports/xhs-latest.json
```

将结果写入待审核队列：

```powershell
& 'D:\Program Files\anaconda3\envs\ai_agent\python.exe' `
  'scripts\import_social_exports.py' `
  'data\imports\xhs-latest.json' `
  --platform xiaohongshu `
  --connector redbook
```

导入器兼容数组、`items`、`feeds`、`notes`、`results` 和常见 MCP
`content[].text` JSON 结构。默认状态是 `pending`，不会直接出现在网站上。

在 `data/social_candidates.json` 中核验标题、摘要、发布时间和链接，将确认过的
记录改为 `"review_status": "reviewed"`，再运行：

```powershell
& 'D:\Program Files\anaconda3\envs\ai_agent\python.exe' 'scripts\social_ingest.py'
```

只有白名单新闻源可以使用 `auto_trusted`；社交平台导入不得跳过人工核验。

## 手工添加内容

也可以在 `data/social_sources.json` 的 `items` 数组中加入经过核验的记录：

```json
{
  "platform": "xiaohongshu",
  "source_type": "public_page",
  "title": "用于网站展示的短标题",
  "summary": "人工整理的短摘要，不复制原文",
  "author": "作者或机构",
  "url": "https://www.xiaohongshu.com/...",
  "published_at": "2026-07-18",
  "place_name": "与 spots.json 一致的地点名称",
  "district": "武昌区",
  "tags": ["亲子", "周末"],
  "review_status": "reviewed"
}
```

若链接缺少标题或摘要，可尝试读取公开 OpenGraph 元数据。脚本会先检查域名
白名单和 `robots.txt`：

```powershell
& 'D:\Program Files\anaconda3\envs\ai_agent\python.exe' `
  'scripts\social_ingest.py' --refresh-metadata
```

## 更新流程

GitHub Actions 每 2 小时执行公开资讯发现和数据更新。本地授权导入建议每
2 至 4 小时运行一次，频率过高会增加平台风控风险。

`scripts/update_all.py` 会先生成社交信息流，再运行热点计算。社交提及仅影响推荐
热度，不覆盖票价、营业时间、地址等官方事实字段。页面同时展示来源检查时间、
最新内容发布时间、待审核数量和连接器状态，避免把“刚检查”误解为“刚发布”。
