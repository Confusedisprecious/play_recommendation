# 社交媒体数据接入

网站通过 `data/social_sources.json` 管理经过审核的公开链接，并由
`scripts/social_ingest.py` 生成前端使用的 `data/social_feed.json`。

## 当前接入方式

- 小红书：支持导入已获授权或人工确认可公开访问的笔记链接；当前示例为官方媒体对小红书活动和家长经验的整理，页面会明确标注“官方媒体整理”。
- 抖音：支持公开内容页、公开搜索页，以及后续获得平台权限后的官方 OpenAPI 数据。
- 微博：预留官方 API 接入，需申请应用并提供合法凭证后启用。

不使用账号 Cookie，不自动登录，不处理验证码，不调用未公开接口，也不复制整篇笔记或下载用户媒体文件。

## 添加内容

在 `data/social_sources.json` 的 `items` 数组中加入一条记录：

```json
{
  "platform": "xiaohongshu",
  "source_type": "public_page",
  "title": "用于网站展示的短标题",
  "summary": "人工整理的短摘要，不复制原文",
  "author": "作者或机构",
  "url": "https://www.xiaohongshu.com/...",
  "published_at": "2026-07-18",
  "place_name": "与 spots.json 一致的地点名称",
  "district": "武昌区",
  "tags": ["亲子", "周末"],
  "review_status": "reviewed"
}
```

随后运行：

```powershell
& 'D:\Program Files\anaconda3\envs\ai_agent\python.exe' 'scripts\social_ingest.py'
```

若链接缺少标题或摘要，可尝试读取公开 OpenGraph 元数据。脚本会先检查域名白名单和 `robots.txt`：

```powershell
& 'D:\Program Files\anaconda3\envs\ai_agent\python.exe' 'scripts\social_ingest.py' --refresh-metadata
```

## 更新流程

`scripts/update_all.py` 会先生成社交信息流，再运行热点计算。社交提及仅影响推荐热度，不覆盖票价、营业时间、地址等官方事实字段。
